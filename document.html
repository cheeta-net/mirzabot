مستندات API درگاه پرداخت ربات

📌 معرفی

این API برای ایجاد و مدیریت پرداخت‌ها در ربات طراحی شده است. با استفاده از این API می‌توانید پرداخت جدید ایجاد کرده و لینک پرداخت دریافت کنید.

🔐 احراز هویت

برای استفاده از API باید API Key در هدر هر درخواست ارسال شود:

api-key: your_api_key_here

💳 ایجاد پرداخت

POST /api/payment/create

Query Parameters:

POST /api/payment/create?chat_id=123456&amount=50000

Response:

{ "status": true, "msg": "Success", "data": { "payment_id": "9f3a5-ab56-d567a42a8a5b", "payment_link": "https://your-domain.com/pay/9f3a5-ab56-d567a42a8a5b" } }

📥 بررسی وضعیت پرداخت

GET /api/payment/{payment_id}

Response:

{ "status": true, "msg": "Payment status is confirmed", "data": { "paid": true } }

🔔 Callback System

بعد از اینکه پرداخت توسط سیستم تأیید شود، یک درخواست POST به آدرس Callback شما ارسال می‌شود.

Callback URL (در تنظیمات فروشگاه)

https://your-domain.com/callback

Payload مثال ارسال شده:

{ "payment_id": "9f3a5-ab56-d567a42a8a5b", "status": "confirmed", "fee": 500, "net_amount": 49500 }

⚙️ نکات مهم

  • API Key را محرمانه نگه دارید
  • تمام درخواست‌ها باید دارای API Key باشند
  • مبلغ پرداخت به تومان است
  • Callback فقط در صورت تایید پرداخت ارسال می‌شود
  • اگر callback URL تنظیم نشده باشد، هیچ درخواستی ارسال نمی‌شود
